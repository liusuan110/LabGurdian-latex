# 第二部分「项目设计方案」撰写策略

> **目标**：在≤5页篇幅内（10页总限 − 封面1页 − 摘要1页 − 目录1页 − 第一部分~1页 − 第三部分~0.5页），用**图文穿插**的方式让评委在 3 分钟内抓住核心亮点。

---

## 一、评委视角：他们在扫什么？

英特尔杯初选评审通常是高校教授 + 企业工程师，阅读量大、时间短。根据往届入围作品共性，评委重点关注：

| 评审关注点 | 对应证据 | 我们应提供的形式 |
|:---|:---|:---|
| **场景真实性** — 是不是在解决真问题？ | 痛点数据 + 使用场景照片 | 第一部分已覆盖 |
| **系统完整性** — 是端到端闭环还是单点demo？ | **系统架构图** | ⭐ 一张大图 |
| **Intel平台绑定** — 是否真正利用了异构算力？ | **硬件映射表/图** | 表格 + 架构图标注 |
| **技术深度** — 有没有自己的算法创新？ | **核心算法流程图** | 流程图 + 公式 |
| **可演示性** — 能不能跑起来？已经做了多少？ | **实测截图/效果图** | GUI截图 + 检测效果图 |
| **工程成熟度** — 代码是写了还是画了？ | 模块划分 + 代码量 | 软件架构图 |
| **可行性** — 进度是否合理？ | 甘特图/里程碑表 | 时间线图 |

**核心结论**：评委是**先看图、再看表、最后扫正文**。一张好的架构图 = 半页文字。

## 四、写作技巧：让评委 3 分钟"读懂"

### 4.1 文字密度控制

| 类型 | 比例 | 说明 |
|:---|:---|:---|
| 图&表 | **40-50%** | 每页至少 1 张图或表 |
| 正文 | **40-50%** | 每段不超过 5 行 |
| 空白 | **10%** | 留出呼吸感 |

**黄金法则**：如果一页全是文字没有图表，评委大概率直接翻页。

### 4.2 "证据链" 写法（评委最爱看的套路）

每个技术点用三段式：
```
WHY  → 为什么需要这个技术？（1句话，点出问题）
WHAT → 我们用了什么方案？（2-3句话 + 图/表）
HOW  → 效果如何？（1句话 + 指标/截图）
```

**示例**：
> **面包板上元件多为倾斜放置**（WHY），传统水平框检测导致严重的 IoU 粘连。
> 我们采用 **YOLOv8-OBB 旋转目标检测**（WHAT），提供旋转角度参数 θ，精确隔离密集元件（见图2-2）。
> 实测在密集布线场景下，OBB 相比 HBB 的 mAP@0.5 **提升约 15 个百分点**（HOW）。

### 4.3 关键词前置

每段开头用**加粗关键词**，让评委扫一眼就知道这段在说什么：

✅ **好的**：
> **五级鲁棒孔洞检测管线。**针对不同光照和面包板型号……

❌ **差的**：
> 针对不同光照和面包板型号的孔洞检测问题，我们设计了五级鲁棒管线……

### 4.4 Intel 平台绑定要反复强调

评委很看重"你的项目是不是非Intel不可？"。在描述每个模块时，都要显式标注硬件映射：

- "视觉感知层 → **iGPU (Arc Graphics)** 实现 FP16 实时推理"
- "逻辑层 → **CPU (P-Core)** 承载 NetworkX 图论计算"
- "认知层 → **NPU (AI Boost)** 低功耗运行 INT4 量化 LLM"

### 4.5 数字 > 形容词

❌ "系统具有很高的识别精度"
✅ "元器件识别 mAP@0.5 达到 80%，端到端延迟 < 200ms"

## 五、当前 LaTeX 文件的改进建议

### 现状问题
1. **全文没有任何图片**——纯文字，信息密度高但可读性差
2. **技术描述偏冗长**——校准器五级管线占了太多篇幅，评委不需要看 Level 1/2/3/4 的函数名
3. **缺少实测截图**——没有展示"已经做出来了"的视觉证据
4. **硬件映射不够突出**——Intel 平台优势在架构图中没有直观体现
5. **无甘特图/技术路线图**——进度安排用纯文字不够直观

### 改进优先级
1. **[最高]** 制作系统总体架构图（图2-1），替代大段架构描述文字
2. **[最高]** 截取 GUI 运行截图（图2-3/2-7），证明工程完成度
3. **[高]** 精简校准器/验证器的代码级描述，提升到算法层面
4. **[高]** 添加甘特图替代进度表格
5. **[中]** 添加 OBB vs HBB 对比图

### 图片目录约定
所有图片放在 `latex/pictures/` 下：
```
pictures/
├── system_architecture.png    -- 系统总体架构图（draw.io 导出）
├── obb_vs_hbb.png             -- OBB vs HBB 对比示意
├── yolo_detection.png         -- YOLO 检测效果实测截图  
├── calibration_pipeline.png   -- 五级校准管线流程图
├── pixel_to_netlist.png       -- 像素到网表流程图
├── llm_fallback.png           -- LLM 离线双引擎架构
├── gui_main.png               -- GUI 主界面截图
├── gui_chat.png               -- 聊天面板截图
├── gantt_chart.png            -- 甘特图 / 技术路线图
└── software_arch.png          -- 软件模块架构图
```

## 七、一句话总结
